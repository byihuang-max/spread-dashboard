## 三层架构

```

```

---

## 使用方式（zs前缀）

**所有择时框架操作都用"zs"前缀，避免和其他场景混淆**

### 触发词

- `zs加因子：XXX` → 分析相关性 → 提示命题 → 自动测试
- `zs测命题：XXX` → 生成框架 → 运行回测 → 分析结果
- `zs优化：XXX` → 分析问题 → 提出方案 → 测试对比
- `zs加标的：XXX` → 修改回测 → 运行测试 → 对比分析
- `zs因子库` → 查询当前所有因子

---

## Musk的工作机制

1. **主动提示相关性**：新因子时告诉你和哪些因子/命题相关
2. **自动关联测试**：把新因子放到对应命题里测试
3. **勾稽关系管理**：理解因子之间、因子和命题之间的关系
4. **记录所有操作**：写入工作日志

---
