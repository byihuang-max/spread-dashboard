# 股票择时模块的最终目标是输出股票敞口值

- 日期：2026-03-12
- 类型：框架定位 / 研究边界
- 标签：股票择时, 股票敞口, 冰点, 强势股, 研究框架

## 最终目标
Roni 明确补充：这条研究线当前只聚焦一个模块目标——**股票择时，并最终输出股票敞口值**。

这意味着当前模块不先讨论“策略适配度”，也不做 FOF 全策略适配总控台。那些未来可以成为更大母框架下的其他子模块。

当前模块的唯一核心问题是：

> 当前股票敞口应该给多少合适？

## 冰点的真实定位
冰点不是这套框架的起点，也不是框架本身。

正确顺序是：
1. 在强势股研究框架里，先基于市场理解写了很多**强逻辑单因子**。
2. 对这些单因子做 IC 检验与有效性验证。
3. 在当前阶段，冰点这条线因为**最好理解、最容易刻画、粗看也最有效**，所以被优先拎出来先做。

因此，冰点应被定义为：

> 股票敞口研究框架中的第一个优先展开命题。

## 正确总结构
正确结构不是“强势股 → 冰点 → 风格过滤”，而是：

**目标：输出股票敞口值**
→ 为达成这个目标，研究一组能刻画市场状态的因子
→ 冰点是当前第一条先跑通的子命题
→ 后面继续补其他命题
→ 最后统一整合成股票敞口输出系统

## 表达原则
后续所有 HTML / README / 研究说明都应遵循以下边界：

- 冰点是当前重点命题，不是全部
- 单因子研究服务于股票敞口目标
- 最终系统输出是股票敞口值，而不是单一策略回测结果
