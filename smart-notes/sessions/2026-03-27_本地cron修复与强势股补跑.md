# 2026-03-27 本地 Cron 修复 & 强势股18:00补跑

## 问题
- FOF 看板上耐心资本和强势股板块停在 25 号，其他模块已到 26 号

## 排查过程

### 1. 本地 cron Python 路径错误
- 本地 crontab 用的是 `/usr/bin/python3`，该环境没有 pandas/numpy/requests
- 正确路径是 `/opt/homebrew/bin/python3`
- 已修复

### 2. 云端数据源延迟
- 强势股依赖 Tushare `limit_list_d`，T日数据约 17:30-18:00 才出
- 耐心资本依赖 15min 大单数据，T日数据要 T+1 才出
- 17:00 跑时这两个模块拿不到当天数据，只能算到 T-1

### 3. 云端 generate_leader_pool.py 路径问题
- 脚本里写的 `~/Desktop/gamt-dashboard/...`，但服务器上没有 Desktop 目录
- 改成相对路径，修复后吐池正常生成

## 修复措施

### 本地 crontab
```
0 17 * * 1-5  完整 update_all.py（/opt/homebrew/bin/python3）
0 18 * * 1-5  补跑 momentum_stock
```

### 云端 crontab
```
08:20  完整更新（盘前）
17:00  完整更新（盘后）
18:00  补跑 momentum_stock（新增）
```

### 耐心资本
- 不需要单独补跑，数据源本身 T+1 延迟
- 第二天 08:20 盘前更新自然会补上

## 关键认知
- `git push` 不影响云端本地数据更新，云端有独立代码和 cron
- `git push` 的价值是同步代码变更、备份、发布
- 小改动可以直接推云（rsync/scp/SSH），不一定要走 GitHub
- 云端 GitHub token 已过期，需要重新生成
