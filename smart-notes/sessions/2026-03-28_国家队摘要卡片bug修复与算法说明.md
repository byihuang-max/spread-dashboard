# 2026-03-28 国家队摘要卡片 bug 修复与算法说明

## 类型：决策 + 技术

## 背景
耐心资本持筹页面新增了5个"国家队"摘要卡片（买入节奏/意图、跨品种广度、相对市场力度、持仓集中度、国家队风格重心），但上线后全部显示"无数据"/"0亿"。

## 根因
`patient_calc.py` 中 `build_market_strength()` 和 `build_concentration()` 两个函数，在读取 `global_daily` 的 indices 结构时，误写了多余的 `.latest` 层级。

实际结构：`global_daily[date]['indices'] = {指数名: result_dict}`
代码写的：`v['latest'].get(...)` — 多了一层 `latest`，导致全部取不到值。

## 修复
- `v['latest'].get(...)` → `v.get(...)`
- `info.get('latest')` → 直接用 `info`

## 同步链路教训
- `sync-to-cloud.sh` 排除了 `*.json *.csv *.html`（设计意图：云端自己跑数据生成产物）
- 但 `patient_capital.json` 是本地重算后需要推到云端的，不会自动同步
- **修复产物（json/csv/html）如果本地改了，需要手动 rsync 推云端**

## 算法口径备忘
- **买入节奏/意图**：连续净买天数 + Z分数 + 浮盈组合判断
- **跨品种广度**：8个指数净买入个数 + 风格桶分布
- **相对市场力度**：|净额|÷(买+卖)×100% 均值 → 分4档
- **持仓集中度**：各指数累计净持仓×最新价加总
- **国家队风格重心**：最大仓位指数的风格桶

## 待优化
- 相对市场力度口径永远偏高（异常bar少→方向一致→接近100%），需改分母或换算法
