# 2026-04-12 经济基本面模块 MCP 落地说明

## 一句话
GAMT 的“宏观与流动性 → 经济基本面”这条月频数据链，当前可以先用 **iFinD MCP** 跑通。

原因很简单：
- direct API 的 EDB 正确入口已经确认是 `https://quantapi.51ifind.com/api/v1/edb_service`
- 但当前账号返回 `-4318`，即 **本月额度超限**
- 而 MCP 的 `search_edb` / `get_edb_data` 已实测可成功返回月频宏观数据

所以现阶段更现实的执行口径是：

> **经济基本面模块先用 MCP 把图做起来，后续若 direct API 的 EDB 恢复，再切回更稳的生产链路。**

---

## 为什么这条线成立
这个模块的核心特点不是高频，而是：
- 大部分指标 **月更**
- 指标数量有限
- 重点是把宏观论证链画清楚，而不是做高频大吞吐

所以 MCP 虽然不适合高频批量扫库，但对这一页是够用的。

## 工程注意点
MCP 的 EDB **要控频**。

已实测：连续批量 `search_edb` 会触发：
- `429`
- `用户请求过于频繁，请稍后重试`

所以更合适的做法是：
- 一次只查少量指标
- 把已确认字段固化到配置/映射表
- 后续更新只做 `get_edb_data`，少做反复搜索

一句话：

> **MCP 适合“低频、少量、月更”的宏观图表链路，不适合高频批量探索式扫库。**
