## 教训：sed 批量替换的陷阱

用 `sed` 批量删 emoji 时，有一行把引号也吃掉了：

```js
// 原始：
title.textContent='🔄 '+(s.module_name||s.module||'刷新中');
// sed 删完变成（坏的）：
title.textContent='+(s.module_name||s.module||'刷新中');
// 正确应该是：
title.textContent=(s.module_name||s.module||'刷新中');
```

这个 JS 语法错误导致整个 `<script>` 块崩溃，所有事件绑定失效——侧边栏点击、tab 切换全部挂掉。

**规则：用 sed 批量替换 emoji 后，必须检查 JS 语法完整性（尤其是引号配对）。**

## 微信小程序讨论

Roni 问过能否做成小程序。结论：
- 技术上可行，但 20+ 模块全部重写渲染逻辑工作量大
- 当前用户只有团队内部，H5 + 浏览器已够用
- 更务实的路径：先做 H5 移动端响应式适配，成本低十倍
- 如果未来有对外展示/LP 需求，再考虑小程序（web-view 壳 + 1-2 个原生页面）
