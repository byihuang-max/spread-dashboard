# 腾讯云自动部署机制

## 问题
推送到GitHub后，云端如何更新？本地修改的模块如何显示在云端？

## 架构

```
本地Mac (开发)
  ↓ git commit
post-commit hook 自动触发
  ↓
sync-to-cloud.sh 执行
  ├─ rsync 同步代码到腾讯云
  └─ 重启 gamt.service 后端服务
  ↓ git push
GitHub 仓库
  ↓ 自动触发
Cloudflare Pages (前端部署)
```

## 两条部署路径

### 1. 前端（HTML/JS/CSS）
- **触发**：git push到GitHub
- **部署**：Cloudflare Pages自动拉取并部署
- **生效**：1-2分钟
- **域名**：dashboard.gamtfof.com

### 2. 后端（Python脚本/数据）
- **触发**：git commit（不是push！）
- **机制**：`.git/hooks/post-commit` 自动调用 `scripts/sync-to-cloud.sh`
- **同步**：rsync到腾讯云 `/home/ubuntu/gamt-dashboard/`
- **重启**：`sudo systemctl restart gamt`
- **生效**：立即

## 关键文件

### post-commit hook
```bash
# .git/hooks/post-commit
#!/bin/bash
SCRIPT_DIR="$(cd "$(dirname "$0")/../.." && pwd)"
bash "$SCRIPT_DIR/scripts/sync-to-cloud.sh" &
```

### sync-to-cloud.sh
- 位置：`scripts/sync-to-cloud.sh`
- 功能：rsync同步 + 重启服务
- 排除：`.git`, `__pycache__`, `venv/`, `_cache/`
- 端口：先试22，失败则试2222

## 常见场景

### 场景1：修改Python脚本（如添加模块）
```bash
# 本地修改 update_all.py
git add server/update_all.py
git commit -m "添加中观景气度模块"
# ↑ commit时自动同步到云端并重启
git push
# ↑ push到GitHub（前端也会更新）
```

### 场景2：只修改前端HTML
```bash
git add index.html
git commit -m "更新前端"
# ↑ 云端后端也会同步（虽然没必要）
git push
# ↑ Cloudflare Pages部署前端
```

### 场景3：手动同步（如果hook失败）
```bash
cd ~/Desktop/gamt-dashboard
bash scripts/sync-to-cloud.sh
```

## 注意事项

1. **commit触发同步，不是push**
   - 即使不push，云端也会更新
   - push只影响GitHub和Cloudflare Pages

2. **云端不是git仓库**
   - `/home/ubuntu/gamt-dashboard/` 没有 `.git/`
   - 纯粹是rsync同步过去的文件

3. **定时任务用最新代码**
   - 云端cron跑的是同步后的最新代码
   - 不用担心代码版本不一致

4. **前端更新需要等Cloudflare**
   - 后端立即生效
   - 前端需要1-2分钟部署

## 相关文件

- `.git/hooks/post-commit` - Git钩子
- `scripts/sync-to-cloud.sh` - 同步脚本
- `/etc/systemd/system/gamt.service` - 云端服务配置
- `server/refresh_server.py` - 云端运行的API服务
