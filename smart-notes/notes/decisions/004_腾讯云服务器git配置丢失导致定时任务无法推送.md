# 腾讯云服务器git配置丢失导致定时任务无法推送

**日期：** 2026-03-16  
**类型：** 故障排查  
**关键词：** git配置、定时任务、数据更新、腾讯云

---

## 问题现象

2026-03-16发现前端数据停留在3月13日，虽然定时任务日志显示每天08:20和17:00都在运行且"✅ 完成"，但数据没有更新到前端。

## 根本原因

**腾讯云服务器上的.git目录丢失了**

从日志可以看到错误：
```
fatal: not a git repository (or any of the parent directories): .git
❌ Git 失败: Command '['git', 'add', '-A']' returned non-zero exit status 128.
```

## 为什么.git目录会丢失

**最可能的原因：初始部署方式问题**

腾讯云服务器最初部署时，可能是：
- 直接从本地复制文件到服务器
- 或者用rsync同步，但没有包含.git目录
- 所以从一开始腾讯云服务器上就没有.git

## 为什么之前没发现

1. 之前都是在本地Mac上开发和push
2. 腾讯云服务器只是运行后端API服务
3. 定时任务虽然会尝试git push，但失败了也不影响服务运行
4. 只是前端数据没有更新，但可能没注意到

## 完整的数据流

**正常流程：**
```
定时任务运行 → 更新数据文件 → git add/commit → git push → GitHub → Cloudflare Pages部署 → 前端显示
```

**出问题时的流程：**
```
定时任务运行 → 更新数据文件 → git push失败 ❌ → GitHub没更新 → 前端显示旧数据
```

**关键发现：**
- 定时任务一直在正常运行
- 数据也在服务器上更新了（文件修改时间是最新的）
- 只是因为git配置丢失，没有推送到GitHub
- Cloudflare Pages从GitHub部署，所以前端看不到更新

## 解决方案

**2026-03-16 19:53 修复步骤：**

1. 用rsync把本地的.git目录同步到腾讯云服务器：
```bash
cd ~/Desktop/gamt-dashboard
rsync -avz --exclude='venv' --exclude='__pycache__' .git/ ubuntu@111.229.129.146:/home/ubuntu/gamt-dashboard/.git/
```

2. 配置git用户信息：
```bash
ssh ubuntu@111.229.129.146 "cd /home/ubuntu/gamt-dashboard && \
  git config user.name 'GAMT Server' && \
  git config user.email 'server@gamtfof.com'"
```

3. 测试git push：
```bash
# 创建测试文件并push
echo 'test' > .git-test
git add .git-test
git commit -m 'test: verify git push'
git push  # ✅ 成功
```

4. 手动运行完整更新：
```bash
cd ~/Desktop/gamt-dashboard
python3 server/update_all.py --no-push
git add -A
git commit -m "auto: 全量更新所有模块数据 (20260316 20:41)"
git push  # ✅ 成功
```

## 验证结果

- ✅ 腾讯云服务器的git已修复
- ✅ 所有18个模块的数据都已更新到3月16日
- ✅ 成功push到GitHub
- ✅ Cloudflare Pages自动部署
- ✅ 前端显示最新数据

## 经验教训

1. **初始部署时要用git clone，不要直接复制文件**
2. **定时任务的错误日志要仔细检查**，不能只看"✅ 完成"
3. **数据更新流程要端到端验证**，不能只看服务器上的文件
4. **关键配置（如.git）要定期检查**

## 预防措施

1. 在定时任务脚本中增加git状态检查
2. 如果git push失败，发送告警通知
3. 定期检查前端数据是否是最新的
4. 考虑在服务器上设置git hook，push失败时自动告警
