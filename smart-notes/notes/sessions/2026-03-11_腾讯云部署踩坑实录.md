# 腾讯云部署 — 踩坑实录与最终方案

**定义:** GAMT 看板从本地 Mac 迁移到腾讯云的完整过程记录，包括所有踩过的坑和最终解决方案。

**相关讨论:**
- 起因：酒店 WiFi/VPN 不稳定，本地 Mac 跑看板频繁掉线
- 目标：24/7 稳定在线，不依赖任何人的本地网络

## 最终架构

```
用户浏览器
    ↓ https://dashboard.gamtfof.com
Cloudflare CDN（全球加速 + HTTPS证书）
    ↓ Cloudflare Tunnel（加密通道）
腾讯云服务器（111.229.129.146）
    ↓ localhost:9876
Python 后端（数据API）+ Nginx（静态文件）
```

**关键点：** Tunnel 是出站连接，不走 80/443 端口，绕过 ICP 备案检测。

## 踩坑记录（重要！）

### 坑1：SSH 频繁断连
- **现象：** 连续 SSH 几次就被断，Connection closed
- **原因：** 腾讯云平台层面的 SSH 防护（不是 fail2ban）
- **解决：** 安装 fail2ban + 白名单整个 IP 段（160.16.0.0/16）
- **教训：** 腾讯云轻量服务器有隐形防护，不要频繁连接

### 坑2：ICP 备案拦截
- **现象：** 域名访问被重定向到腾讯云备案提示页
- **原因：** 中国大陆服务器 + 未备案域名 = 必须备案
- **尝试过的方案：**
  - A 记录 DNS only（灰色云朵）→ 被拦
  - A 记录 Proxied（橙色云朵）→ 被拦（腾讯云检测 Host 头）
  - Cloudflare SSL Full → 525 错误
  - Cloudflare SSL Flexible → 还是被备案页拦截
  - IP 直连 → 能用但没 HTTPS
- **最终方案：** Cloudflare Tunnel（不走 80/443，从内部出站）

### 坑3：index.html 被覆盖
- **现象：** 看板显示 301 Moved Permanently
- **原因：** 在网站根目录执行 `curl -sO` 下载文件，覆盖了 index.html
- **教训：** 永远不要在网站根目录执行 curl -sO

### 坑4：Cloudflare SSL 模式
- **Full 模式：** CF 用 HTTPS 连源站，源站证书必须有效 → 腾讯云拦了 443 端口
- **Flexible 模式：** CF 用 HTTP 连源站 → 如果 Nginx 有 HTTP→HTTPS 重定向会死循环
- **Tunnel 模式：** 不需要关心 SSL 模式，Tunnel 自带加密

### 坑5：出口 IP 会变
- **现象：** Mac 的公网 IP 从 160.16.148.207 变成 160.16.56.12
- **影响：** fail2ban 白名单失效
- **解决：** 白名单用 CIDR 网段（160.16.0.0/16）而不是单个 IP

## 决策记录

- **为什么选腾讯云而不是阿里云？** Roni 已有腾讯云账号
- **为什么不做备案？** 当前 Tunnel 方案完全可用，备案周期长且需要转域名
- **为什么盘后改到17:00？** 15:35太早，数据源还没刷新完
- **为什么用 Tunnel 而不是 Nginx 直连？** ICP 备案问题，Tunnel 绕过检测
- **为什么 SSL 用 Flexible？** Tunnel 模式下实际不经过 SSL 层，Flexible 避免冲突
