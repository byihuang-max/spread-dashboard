# 腾讯云部署 - GAMT 看板迁移指南

**定义:** 将 GAMT 投研看板从本地 Mac mini 迁移到腾讯云轻量应用服务器，实现 24/7 稳定在线，不再依赖本地网络。

**相关讨论:**
- 解决酒店 WiFi/VPN 不稳定导致看板频繁掉线的问题
- 原架构：Mac mini → Cloudflare Tunnel → 用户访问
- 新架构：腾讯云（公网 IP 直连）→ 用户访问

## 架构变化

### 之前（本地）
```
Mac mini（跑看板 + 数据刷新）
    ↓
Cloudflare Tunnel（穿透内网）
    ↓
dashboard.gamtfof.com
    ↓
用户访问
```
**问题：** WiFi 断 / VPN 掉 / Mac 关机 → 看板全挂

### 现在（云端）
```
腾讯云服务器（24/7 运行）
    ↓
Nginx 反代 + Let's Encrypt HTTPS
    ↓
dashboard.gamtfof.com（A 记录直指云 IP）
    ↓
用户访问
```
**优势：** 不依赖任何人的网络，服务器重启后自动恢复

## 技术栈

- **Web 服务器:** Nginx（静态文件 + API 反代）
- **后端:** Python 3.12 + venv（refresh_server.py 监听 9876 端口）
- **进程管理:** systemd `gamt.service`（开机自启 + 崩溃自动重启）
- **HTTPS:** Let's Encrypt 证书（到期 2026-06-09，certbot 自动续期）
- **数据刷新:** cron 定时任务

## 定时刷新计划

| 时间 | 频率 | 说明 |
|---|---|---|
| 08:20 | 周一至周五 | 盘前刷新，开盘前准备好数据 |
| 17:00 | 周一至周五 | 盘后刷新，收盘后等数据源更新完毕再拉取 |

## 运维指南

### 常用命令（SSH 登录后）
```bash
# 查看服务状态
sudo systemctl status gamt

# 重启服务
sudo systemctl restart gamt

# 查看服务日志
sudo journalctl -u gamt -n 50 --no-pager

# 查看 cron 刷新日志
tail -100 /tmp/gamt.log

# 手动触发全量刷新
cd ~/gamt-dashboard && venv/bin/python3 server/update_all.py
```

### 代码更新流程
从 Mac mini 同步最新代码到云端：
```bash
rsync -az --exclude='.git' --exclude='__pycache__' \
  ~/Desktop/gamt-dashboard/ ubuntu@111.229.129.146:~/gamt-dashboard/
```

### 安全注意事项
1. 腾讯云账号建议开启 MFA 二次验证
2. 服务器只开放 22/80/443 三个端口
3. 看板数据均为公开市场数据，无核心机密
4. 用户密码加盐哈希存储（与本地版一致）

## 决策记录

- **为什么选轻量服务器而不是 CVM？** 便宜，够用，自带公网 IP
- **为什么不继续用 Cloudflare Tunnel？** 根本问题是本地网络不稳定，Tunnel 只是中间层
- **为什么关掉 Cloudflare 代理？** 用自己的 Let's Encrypt 证书更简单，不需要 Cloudflare 的 CDN
- **为什么盘后改到 17:00？** 15:35 太早，很多数据源还没刷新完
