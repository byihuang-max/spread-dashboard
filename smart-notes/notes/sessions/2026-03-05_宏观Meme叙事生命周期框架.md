## 核心定义厘清

### 这个系统实际在做什么

**本质：舆情频次 → 事件热度时序表**

```
每天拉财经新闻
    ↓
统计每个主题词出现频次
    ↓
频次高 = 热度高 = 打分高
    ↓
每天记录一次 → 形成时序热度表
```

"叙事得分7/10" 的真实含义：**今天这个主题在财经信息里出现密度很高**。

## 题材交易 vs 宏观 Meme 交易

| 维度 | 题材交易 | 宏观 Meme 交易 |
|------|---------|--------------|
| 时间尺度 | 1–5天 | 数周到数月 |
| 信号特征 | 短促尖峰，快速衰减 | 缓慢积累，持续高位 |
| 核心问题 | 今天爆了什么？ | 这个叙事是否正在成为共识？ |
| 机会判断 | 抢在新闻消化前进入 | 抢在市场完全定价前配置 |
| 事件波适用性 | ✅ 完全适用 | ❌ 时间尺度太短 |

## 宏观 Meme 生命周期四阶段

每个阶段的判断维度：
1. **历史分位** — 当前热度在自身历史中的位置（高=成熟，低=新兴）
2. **动量** — MA7斜率（上升=早期/积累，下降=消退）
3. **持续时长** — 连续活跃天数（短=新兴，长=成熟/共识）
4. **价格确认** — 叙事热但价格未动=机会窗口；价格已动=共识化

```
🌱 新兴  → 热度从低位升起，市场尚未定价，最大配置窗口
🔥 积累  → 持续升温，价格开始跟随，窗口收窄中  
⚡ 共识  → 高位稳定，已成市场共识，慎追
📉 消退  → 热度下行，叙事衰退，关注反向机会
```

**关键判断：新兴叙事 vs 共识叙事**  
一旦成为共识 → 对应资产已充分定价 → 宏观 Meme 交易窗口关闭

## 完整系统架构（三层）

```
Layer 1: 舆情热度层
  narrative_monitor.py → 每日得分 + narrative_history.json

Layer 2: 价格背离检测层  
  get_divergence_signal() → 叙事热但价格未动 = 机会信号

Layer 3: 宏观生命周期层（本次新增）
  macro_lifecycle.py → 多周趋势 → 阶段判断 → 配置建议
```
