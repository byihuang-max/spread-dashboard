## 因子 01：涨跌家数比

### 频率
- 日频

### 类别
- 广度

### 因子定义
- 上涨家数与下跌家数的对比，用于衡量市场整体涨跌分布状态。
- 第一版建议口径：
- `涨跌家数比 = (上涨家数 - 下跌家数) / (上涨家数 + 下跌家数)`

### 逻辑说明
- 这是一个强逻辑因子。
- 如果市场处于明显强势或明显弱势状态，广度一定会体现出来。
- 当上涨家数显著多于下跌家数，说明市场正在普遍修复，风险偏好在扩散。
- 当下跌家数显著多于上涨家数，说明市场仍处于普遍风险回避状态。
- 这个因子不主要负责确认高潮，而是负责判断市场底板有没有抬起来。

### 极端含义
- 极强：市场处于普涨、情绪扩散、风险偏好显著抬升。
- 极弱：市场处于普跌、风险释放、亏钱效应广谱蔓延。

### 在情绪链条中的角色
- 先导

### 与原有因子的关系
- 前置
- 它补的是原有涨停家数、跌停家数、晋级率、封单强度之前的底层广度。

### 是否纳入核心观察
- 是

### 备注
- 后续适合做环境过滤器，也适合和首板数量、晋级率做领先关系验证。
- 暂不急于单独做交易信号回测。

## 盘中发现层母框架（角色层 / 事件层 / 叙事层）

- 定位：盘中发现层不是简单“再补几个因子”，而是把人工盘面语言翻译成 PM 可讨论、可计算的状态框架。
- 结构：`角色层 → 事件层 → 叙事层`。
- 与其他层的关系：
  - 日频因子负责给出 **T日收盘先验**
  - 盘中发现层负责在 **T+1 交易时段** 对这个先验做确认、强化或证伪
  - 合成因子 / 阶段输出负责把这些信息翻成阶段判断和仓位动作

### 一、角色层：盘中到底在看谁

- 龙头观察池：由 T 日收盘后的日频强信号筛出来的候选序列，不是一只固定股票。
- 辨识度龙头：市场最认、位置最高、最有带动性的核心票。
- 成交额龙头：当日成交额最高、最吸金的核心票。
- 龙一 / 龙二 / 龙三：同一条主线或同一梯队中，承担不同位置功能的动态角色。
- 高位核心：高板、高辨识度、高情绪杀伤力的个股集合。
- 首板试错：当天最先承接风险偏好的低位启动组。
- 跟风扩散：从单核走向多核、从单票走向板块共振的跟随层。
- 跌停翘板修复：从极端亏钱效应反向修复风险偏好的角色集合。

### 龙头定义（当前正式口径）

#### 1）龙头观察池
- 不是单票定义，而是 T 日收盘后由日频信号筛出来的观察池。
- 进入观察池的优先条件：
  - 高板 / 最高板
  - 打板成交额高
  - 高度高
  - 在日频强信号中排名靠前
- 含义：T 日的日频强因子，到 T+1 就转化为“龙头观察池”。

#### 2）辨识度龙头
- 在观察池中，市场最认、最有高度、最有位置感、最有带动性的那只。
- 更偏人工交易语义。

#### 3）成交额龙头
- 在观察池中，当日成交额最高的核心票。
- 更偏量价资金语义。

#### 4）为什么要拆成两类龙头
- A 股里“龙头 / 龙一 / 龙二 / 龙三”是动态变化的。
- 辨识度龙头和成交额龙头有时重合，有时不重合。
- 人工盘面处理经常正是在观察它们是否联动、是否切换、是否共振。

### 二、事件层：盘中到底发生了什么

- 首次封板
- 炸板
- 回封
- 翘板
- 尾盘回落
- 10cm → 20cm 扩散
- 单核 → 多核扩散
- 高位股集体松动
- 梯队同步加强
- 午后分歧后修复

### 三、叙事层：这些事件意味着什么

- 龙头确认
- 试错启动
- 梯队加强
- 高度突破
- 板块共振扩散
- 分歧转修复
- 高位退潮
- 风险偏好抬升 / 回落

## 盘中发现因子 v1（单独作为盘中确认层）

- 定义：盘中因子 = 在交易时段内出现的、用于确认或证伪当日日频判断的事件型 / 过程型信号。
- 页面位置：Tab B「盘中发现因子」。
- 统一要求：每个盘中因子都必须回答 3 个问题：
  - 它挂在哪个角色上
  - 它记录了什么事件
  - 它对应什么叙事含义

### 推荐字段
- `name`
- `display_name`
- `factor_level` = `intraday_signal`
- `frequency` = `intraday`
- `role_layer` = `character`
- `event_layer` = `event / process`
- `narrative_layer`
- `trigger_type` = `event / process / confirm / warning`
- `observation_window` = `open_30m / morning / late_morning / afternoon / close_near / full_session`
- `role_in_chain`
- `output_type` = `binary / count / ratio / timestamp / state_label`
- `related_daily_factors`
- `impact_on_stage`
- `logic`
- `status`
