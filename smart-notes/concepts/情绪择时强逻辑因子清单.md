## 因子 01：涨跌家数比

### 频率
- 日频

### 类别
- 广度

### 因子定义
- 上涨家数与下跌家数的对比，用于衡量市场整体涨跌分布状态。
- 第一版建议口径：
- `涨跌家数比 = (上涨家数 - 下跌家数) / (上涨家数 + 下跌家数)`

### 逻辑说明
- 这是一个强逻辑因子。
- 如果市场处于明显强势或明显弱势状态，广度一定会体现出来。
- 当上涨家数显著多于下跌家数，说明市场正在普遍修复，风险偏好在扩散。
- 当下跌家数显著多于上涨家数，说明市场仍处于普遍风险回避状态。
- 这个因子不主要负责确认高潮，而是负责判断市场底板有没有抬起来。

### 极端含义
- 极强：市场处于普涨、情绪扩散、风险偏好显著抬升。
- 极弱：市场处于普跌、风险释放、亏钱效应广谱蔓延。

### 在情绪链条中的角色
- 先导

### 与原有因子的关系
- 前置
- 它补的是原有涨停家数、跌停家数、晋级率、封单强度之前的底层广度。

### 是否纳入核心观察
- 是

### 备注
- 后续适合做环境过滤器，也适合和首板数量、晋级率做领先关系验证。
- 暂不急于单独做交易信号回测。

## 盘中发现因子 v1（单独作为盘中确认层）

- 定位：不与日频原始因子混放，也不直接等于最终结论。
- 定义：盘中因子 = 在交易时段内出现的、用于确认或证伪当日日频判断的事件型 / 过程型信号。
- 页面位置：Tab B「盘中发现因子」。
- 与其他层的关系：日频负责底层先验，盘中负责当日确认/证伪，阶段输出负责状态归纳。

### 推荐字段
- `name`
- `display_name`
- `factor_level` = `intraday_signal`
- `frequency` = `intraday`
- `trigger_type` = `event / process / confirm / warning`
- `observation_window` = `open_30m / morning / late_morning / afternoon / close_near / full_session`
- `role_in_chain`
- `output_type` = `binary / count / ratio / timestamp / state_label`
- `related_daily_factors`
- `impact_on_stage`
- `logic`
- `status`
