# HPM因子库框架解析：双层选股架构

> 来源：某PM提供的外部因子包，2026-04-27 分析
> 分类：conversations
> 标签：因子库, 选股框架, 风控剔除, HPM

## 背景

收到一个PM提供的HPM因子库（黑盒，只有因子值，无算法），包含7个Alpha因子 + 4个事件因子，总数据量约720MB，覆盖全A股。

## 核心架构：两层过滤

```
全A股票池 (~5200只)
    │
    ▼
┌─────────────────────────┐
│  第一层：Alpha因子打分     │
│  7个因子综合排序/加权      │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│  第二层：事件风险剔除      │
│  4个负面事件标记，命中即剔除 │
└───────────┬─────────────┘
            │
            ▼
      最终选股结果
  (Alpha高分 且 无风险标记)
```

## 关键设计特点

1. **两种使用节奏**：大部分因子T+1开盘用，`new_rec`和`recur`当日即可用（可能盘中更新）
2. **两种数据格式**：5个标准格式 + 2个旧格式（id/Stkcd/statistical_date/factor_value/mark）
3. **mf_base是核心**：历史最长（7年）、数据量最大，其他因子是增量补充
4. **事件因子全二值**：不参与打分，只做负面筛除
5. **黑盒交付**：只有因子值结果，无构造算法

## 可借鉴的框架思路

- **Alpha层 + 剔除层**的双层结构，清晰分离"选什么"和"不选什么"
- 因子按频率分类（日频/周频），使用时点明确（T+1开盘 vs 当日）
- 事件因子作为独立风控层，与alpha打分解耦
- 因子信息表（xlsx）作为元数据管理，记录频率、覆盖、使用方式
