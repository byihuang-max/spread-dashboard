## 背景

Roni在查看量化宽基Alpha环境评分时，发现了两个问题：

1. **流动性评分滞后**
   - 市场从3.2万亿decay到2.5万亿（-22%）
   - 但评分显示2.5万亿=90分（过热）
   - 无法捕捉"相对于近期常态的缩量"

2. **Alpha环境评分矛盾**
   - 流动性52分（偏低）
   - 但Alpha环境显示90分（极佳）
   - 逻辑不合理

## 优化一：流动性评分 - 加大周度环比权重

### 问题分析

**Roni的观察：**
> "之前都差不多是3.2万亿以上的成交额，最近都掉到2.5万亿成交，其实decay了很多。"

> "我估计现在A股量化成交量占比6成以上了，也就是说2.4万亿里面大多数都是量化，和9.24之前都跌到过7-8千亿，我想表达的意思是现在2万亿可能就是个基础。"

**核心问题：**
1. 绝对值阈值有滞后性
2. 市场结构变化（量化占比6成+）
3. 需要捕捉相对变化，而不只是绝对值

### 优化方案

**方案选择：** 方案2 - 降低绝对值权重，提高环比权重

**具体改动：**

#### 1. 基础分（绝对值）
```
旧版：10-90分（80分权重）
新版：20-80分（60分权重）

阈值上移：
< 12000亿 → 20分（+4000亿）
12000-15000亿 → 40分（+5000亿）
15000-18000亿 → 55分（+6000亿）
18000-22000亿 → 65分（+7000亿）
22000-28000亿 → 75分（+8000亿）
> 28000亿 → 80分（上限降低）
```

#### 2. 周度环比修正
```
旧版：-20~+10分（30分权重）
新版：-40~+20分（60分权重）

< -20%  → -40分（暴跌式缩量）
-20~-15% → -30分（急剧缩量）
-15~-10% → -20分（明显缩量）
-10~-5% → -10分（温和缩量）
+10~+15% → +10分（明显放量）
+15~+20% → +15分（急剧放量）
> +20%  → +20分（暴涨式放量）
```

### 效果对比

**实际案例（2026-03-15）：**
- 成交额：24588亿
- 趋势：缩量

**v1评分：**
- 基础分：90分（>20000亿）
- 环比修正：约-10分
- 最终：69分（正常）🟡

**v2评分：**
- 基础分：75分（22000-28000亿）
- 环比修正：约-15分
- 最终：52分（偏低）🟠

**差异：-17分，评级从"正常"降到"偏低"**

### 设计哲学

**关键洞察：**
- 2.5万亿在牛市=缩量（危险）
- 2.5万亿在熊市=过热（也危险）
- 要看相对于近期常态的变化，而不是绝对值

**权重平衡：**
- 绝对值和环比各60分
- 环比可以大幅改变评级
- 更准确反映市场状态

## 优化二：Alpha环境评分 - 流动性母项逻辑

### 问题分析

**Roni的洞察：**
> "我的理解是流动性是阿尔法的母项，剩下是子项，但是阿尔法的环境肯定是要包括母项的打分的。现在的逻辑是母项不报警，子项独立计算后给出了阿尔法环境极佳其实是不对的，应该是母项给大的权重，子项顶多是个系数。"

**矛盾现象：**
- 流动性：52分（偏低）🟠
- Alpha环境：90分（极佳）🟢
- 综合评分：74分（均衡）🟡

**根本问题：**
- 旧逻辑：流动性25% + alpha_env50%
- alpha_env独立计算，不考虑流动性
- 流动性52分被alpha_env 90分拉高

### 优化方案

**核心思想：**
- 流动性 = 母项（70%权重）
- 子项 = 修正系数（30%权重）

**新逻辑：**
```python
# Step 1: 流动性门槛
if 流动性 < 30:
    alpha_env = 15  # 枯竭直接红灯

# Step 2: 流动性主导（70%）
liq_base = 流动性分数

# Step 3: 子项修正（30%）
sub_score = f(离散度, 风格集中度)
mkt_adj = 市场预期修正
micro_adj = 微观结构修正

# Step 4: Alpha环境综合
alpha_env = liq_base × 0.7 + sub_score × 0.3 + mkt_adj + micro_adj
```

### 效果对比

**实际案例（2026-03-15）：**
- 流动性：52分
- 子项：90分（离散度高+风格分散）

**旧逻辑：**
```
alpha_env = 90分（独立计算）
综合评分 = 52×25% + 90×50% + ... = 74分
```
- Alpha环境：90分（极佳）🟢
- 矛盾：流动性偏低，但alpha极佳

**新逻辑：**
```
alpha_env = 52×0.7 + 90×0.3 = 63分
```
- Alpha环境：63分（良好）🟡
- 合理：流动性偏低，alpha良好（不是极佳）

**差异：Alpha环境从90分降到63分（-27分）**

### 设计哲学

**为什么流动性是母项？**
1. 生存基础：没有流动性，选股再好也无法执行
2. 前提条件：流动性决定"能不能做"
3. 主导因子：子项决定"做得好不好"

**为什么70% vs 30%？**
1. 70%足够主导，体现母项地位
2. 30%足够修正，反映子项影响
3. 不是50/50（无法体现主次）
4. 不是80/20（子项影响太小）

### 双重优化后的评分

**2026-03-15实际数据：**
- 成交额：24588亿
- 趋势：缩量
- 离散度：90分（高）
- 风格集中度：90分（分散稳定）

**最终评分：**

- 综合评分：63分（均衡）🟡

**诊断：**
> 流动性偏低(24588亿，温和缩量，波动偏大) → 离散度高+风格分散稳定，alpha空间充裕 → 市场预期偏乐观(IM分位80%) → 微观健康

**核心改进：**

## 后续优化方向

### 流动性评分

1. **动态阈值**
   - 根据近期均值自动调整阈值
   - 而不是固定的2万亿、2.8万亿

2. **分市场状态**
   - 牛市/熊市用不同的阈值
   - 或者用波动率调整权重

3. **量化占比监控**
   - 实时监控量化成交占比
   - 动态调整基础流动性需求

### Alpha环境评分

1. **动态权重**
   - 流动性充裕时，子项权重提高（60/40）
   - 流动性偏低时，子项权重降低（80/20）

2. **非线性关系**
   - 流动性偏低 → 子项打折（更激进）
   - 流动性充裕 → 子项加成

3. **阈值优化**
   - 流动性多少分算"偏低"？
   - 流动性多少分算"充裕"？
   - 不同阈值对应不同的权重分配
