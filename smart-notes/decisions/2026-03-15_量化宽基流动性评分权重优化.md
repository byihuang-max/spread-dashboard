# 量化宽基阿尔法环境 - 流动性评分权重优化记录

**日期：** 2026-03-15  
**类型：** 决策记录 - 算法优化  
**关键词：** 量化宽基、流动性评分、周度环比、权重调整

---

## 背景

### 问题发现

Roni观察到：
- 之前市场常态：3.2万亿成交额
- 现在市场状态：2.5万亿成交额
- **实际变化：** decay了22%（明显缩量）
- **旧算法评分：** 2.5万亿 → 90分（过热）❌

**矛盾：** 绝对值看起来很高，但相对于近期常态是缩量，旧算法无法捕捉这种相对变化。

### 市场结构变化

- **9.24之前：** 量化占比低，7-8千亿是底部
- **9.24之后：** 量化占比6成以上
- **新常态：** 2万亿里1.2万亿是量化，基础流动性需求提高

---

## 优化方案

### 方案选择

讨论了3个方案：
1. 扩大环比修正范围
2. **降低绝对值权重，提高环比权重** ✅（采用）
3. 环比作为乘数

**选择理由：** 方案2平衡了绝对值和相对变化，既保留了基础判断，又能捕捉趋势变化。

---

## 具体改动

### 一、基础分（绝对值）

**旧版（v1）：**
```
< 8000亿  → 10分（枯竭）
8000-10000亿 → 30分（偏低）
10000-12000亿 → 50分（一般）
12000-15000亿 → 65分（正常）
15000-20000亿 → 80分（充裕）
> 20000亿 → 90分（过热）

权重：80分（10-90）
```

**新版（v2）：**
```
< 12000亿 → 20分（枯竭）+4000亿
12000-15000亿 → 40分（偏低）+5000亿
15000-18000亿 → 55分（一般）+6000亿
18000-22000亿 → 65分（正常）+7000亿
22000-28000亿 → 75分（充裕）+8000亿
> 28000亿 → 80分（过热）上限降低

权重：60分（20-80）
```

**变化：**
- 阈值整体上移50-100%（反映量化占比提升）
- 上限降低90→80（为环比修正留出空间）
- 权重降低80→60

### 二、周度环比修正

**旧版（v1）：**
```
< -15%  → -20分（急剧缩量）
-15~-10% → -15分（明显缩量）
-10~-5% → -8分（温和缩量）
+10~+15% → +5分（明显放量）
> +15%  → +10分（急剧放量）

权重：30分（-20~+10）
```

**新版（v2）：**
```
< -20%  → -40分（暴跌式缩量）⚠️
-20~-15% → -30分（急剧缩量）
-15~-10% → -20分（明显缩量）
-10~-5% → -10分（温和缩量）
+10~+15% → +10分（明显放量）
+15~+20% → +15分（急剧放量）
> +20%  → +20分（暴涨式放量）

权重：60分（-40~+20）
```

**变化：**
- 权重翻倍：30→60
- 扣分/加分范围扩大
- 与绝对值权重相等（各60分）

### 三、其他修正（不变）

- 稳定性（CV）：-15~0分
- 趋势：-5~+5分
- 脉冲：-5分

**市场状态：**

**差异：-17分，评级从"正常"降到"偏低"**

## 设计哲学

### v1的问题

1. **绝对值权重过高**（80分）
   - 环比修正有限（30分）
   - 无法大幅改变评级

2. **阈值滞后**
   - 基于旧市场结构（量化占比低）
   - 2万亿被认为是"过热"，但现在是新常态

### v2的改进

1. **权重平衡**
   - 绝对值和环比各60分
   - 环比可以大幅改变评级

2. **阈值上移**
   - 反映量化占比6成的新常态
   - 2万亿是"一般"，不是"过热"

3. **捕捉相对变化**
   - 不只看绝对值，更看相对于近期常态的变化
   - 2.5万亿在牛市=缩量，在熊市=过热

**核心逻辑：**
- 市场结构变了，评分标准也要变
- 绝对值有滞后性，相对变化更重要
- 量化占比提升 → 基础流动性需求提高

## 后续优化方向

1. **动态阈值**
   - 根据近期均值自动调整阈值
   - 而不是固定的2万亿、2.8万亿

2. **分市场状态**
   - 牛市/熊市用不同的阈值
   - 或者用波动率调整权重

3. **量化占比监控**
   - 实时监控量化成交占比
   - 动态调整基础流动性需求
