# Alpha环境评分逻辑优化 - 流动性母项权重调整

**日期：** 2026-03-15  
**类型：** 决策记录 - 逻辑修正  
**关键词：** Alpha环境、流动性母项、权重调整

---

## 问题发现

### 矛盾现象

**市场状态：**
- 流动性：52分（偏低）🟠
- 离散度：90分（高）🟢
- 风格集中度：90分（分散稳定）🟢

**旧逻辑输出：**
- Alpha环境：90分（极佳）🟢
- 综合评分：74分（均衡）🟡

**矛盾：** 流动性偏低，但Alpha环境显示"极佳"

### Roni的洞察

> "我的理解是流动性是阿尔法的母项，剩下是子项，但是阿尔法的环境肯定是要包括母项的打分的。现在的逻辑是母项不报警，子项独立计算后给出了阿尔法环境极佳其实是不对的，应该是母项给大的权重，子项顶多是个系数。"

**核心问题：**
- 流动性应该是**母项**（前提和主导）
- 其他因子应该是**子项**（修正系数）
- 不能让子项的高分掩盖母项的低分

---

## 旧逻辑分析

### 旧权重结构

```python
# Step 1: 流动性门槛（枯竭直接出局）
if liq['score'] < 30:
    return 红灯

# Step 2: 离散度 × 风格集中度 → alpha_env（独立计算）
if 离散度高 and 风格分散:
    alpha_env = 90  # 极佳

# Step 3: 综合评分（加权平均）
env_score = 流动性25% + alpha_env50% + 市场预期10% + 微观15%
```

**问题：**
1. **流动性权重过低**（25%）
   - alpha_env占50%，主导了最终评分
   - 流动性52分被alpha_env 90分拉高

2. **alpha_env独立计算**
   - 只看离散度和风格集中度
   - 不考虑流动性的影响
   - 结果：流动性偏低，但alpha_env显示"极佳"

3. **逻辑不符合现实**
   - 流动性是量化策略的生存基础
   - 没有流动性，选股再好也无法执行
   - 流动性应该是主导因子，不是25%的配角

---

## 新逻辑设计

### 核心思想

**流动性 = 母项（70%）**
- 前提条件：枯竭直接出局
- 主导因子：决定Alpha环境的基础分
- 大权重：70%

**子项 = 修正系数（30%）**
- 离散度 × 风格集中度：选股空间
- 市场预期：虹吸风险
- 微观结构：踩踏风险
- 小权重：30%

### 新权重结构

```python
# Step 1: 流动性门槛（枯竭直接出局）
if liq['score'] < 30:
    alpha_env = 15
    return 红灯

# Step 2: 流动性主导（70%权重）
liq_base = liq['score']  # 流动性作为基础分

# Step 3: 子项修正（30%权重）
# 3.1 离散度 × 风格集中度
if 离散度高 and 风格分散:
    sub_score = 90
elif 离散度高 but 风格集中:
    sub_score = 55
elif 离散度低:
    sub_score = 25
else:
    sub_score = 离散度60% + 风格集中度40%

# 3.2 市场预期修正
if 大票虹吸确认:
    mkt_adj = -15
elif 极端贴水:
    mkt_adj = -3
else:
    mkt_adj = 0

# 3.3 微观尾部叠加
if 踩踏预警:
    micro_adj = -12
elif 有压力:
    micro_adj = -5
else:
    micro_adj = 0

# Step 4: Alpha环境综合评分
alpha_env = liq_base × 0.7 + sub_score × 0.3 + mkt_adj + micro_adj
```

## 效果对比

### 实际案例（2026-03-15）

**市场状态：**
- 流动性：52分（偏低）
- 子项：90分（离散度高+风格分散）
- 市场预期：无虹吸
- 微观结构：健康

**旧逻辑：**
```
alpha_env = 90分（独立计算，只看子项）
综合评分 = 52×25% + 90×50% + 55×10% + 70×15%
        = 13 + 45 + 5.5 + 10.5
        = 74分（均衡）
```
- Alpha环境：90分（极佳）🟢
- 综合评分：74分（均衡）🟡
- **矛盾：流动性偏低，但alpha极佳**

**新逻辑：**
```
alpha_env = 52×0.7 + 90×0.3 + 0 + 0
         = 36.4 + 27 + 0 + 0
         = 63.4 ≈ 63分
```
- Alpha环境：63分（良好）🟡
- 综合评分：63分（均衡）🟡
- **合理：流动性偏低，alpha良好（不是极佳）**

**差异：**
- Alpha环境：90分 → 63分（-27分）
- 综合评分：74分 → 63分（-11分）
- 评级：极佳 → 良好

## 设计哲学

### 为什么流动性是母项？

1. **生存基础**
   - 量化策略需要频繁交易
   - 没有流动性，选股再好也无法执行
   - 滑点、冲击成本会吃掉超额

2. **前提条件**
   - 流动性枯竭 → 直接红灯
   - 流动性偏低 → 限制了alpha的发挥空间
   - 流动性充裕 → alpha才能充分释放

3. **主导因子**
   - 流动性决定了"能不能做"
   - 子项决定了"做得好不好"
   - 先有"能做"，才有"做好"

### 为什么子项是系数？

1. **修正作用**
   - 离散度高 → 选股空间大 → 加分
   - 风格集中 → alpha不稳 → 扣分
   - 虹吸确认 → 超额承压 → 扣分

2. **不能主导**
   - 子项再好，流动性不够也白搭
   - 子项只能在流动性允许的范围内发挥作用
   - 30%权重足够反映子项的影响

3. **交互关系**
   - 流动性充裕 + 子项好 → 极佳
   - 流动性充裕 + 子项差 → 一般
   - 流动性偏低 + 子项好 → 良好（不是极佳）
   - 流动性偏低 + 子项差 → 困难

## 权重分配逻辑

### 70% vs 30% 的理由

**流动性70%：**
- 主导因子，决定基础分
- 反映"能不能做"的问题
- 权重应该大于50%，才能真正主导

**子项30%：**
- 修正因子，调整基础分
- 反映"做得好不好"的问题
- 权重不能太小（否则没意义），也不能太大（否则喧宾夺主）

**为什么不是50% vs 50%？**
- 50/50会让流动性和子项平分秋色
- 无法体现流动性的"母项"地位
- 70/30更能体现主次关系

**为什么不是80% vs 20%？**
- 80/20会让子项影响太小
- 离散度、风格集中度等因子确实重要
- 30%权重足够反映它们的影响

## 后续优化方向

1. **动态权重**
   - 流动性充裕时，子项权重可以提高（比如60/40）
   - 流动性偏低时，子项权重应该降低（比如80/20）
   - 根据流动性水平动态调整权重

2. **非线性关系**
   - 流动性枯竭 → 直接红灯（已实现）
   - 流动性偏低 → 子项打折（可以更激进）
   - 流动性充裕 → 子项加成

3. **阈值优化**
   - 流动性多少分算"偏低"？（现在是<55）
   - 流动性多少分算"充裕"？（现在是>75）
   - 不同阈值对应不同的权重分配
