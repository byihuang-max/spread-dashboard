这份报告的母体不是“冰点策略”，而是更大的 择时框架（Timing Framework）。

整个框架的核心逻辑是：
情绪 / 资金 / 风格 / 风险状态 → 状态识别 → 指数映射与执行动作

框架图：输入层 → 状态层 → 动作层

这个图想表达的是：择时不是单点预测，而是一个分层系统。
先从输入变量里识别“市场现在处于什么状态”，再把状态映射成真正可执行的动作。

框架输入层

- 涨停家数 / 跌停家数
- 封单额轧差 / 封板质量 / 晋级率
- 综合情绪 / 炸板率 / 连板高度
- 风格相对强弱（300 / 500 / 1000 / 2000）
- 风险状态（risk-on / risk-off）

中间状态层

- 冰点 / 过热 / 中性
- small-cap / large-cap / neutral
- 情绪修复 / 情绪退潮
- 风险开 / 风险关

输出动作层

- 做不做
- 做哪个指数（300 / 500 / 2000）
- 做多久（3 / 5 / 10 / 15 / 20天）
- 未来可扩展到：仓位 / 分批建仓 / 动态持有期

V1.1 结论

- `2000-300` 这个 proxy 是有信息量的
- small-only 过滤后，胜率 / 单笔收益 / 回撤控制都更好
- 但 `large -> 做300` 当前并没有跑赢 baseline

所以 V1.1 得出的核心判断是：
过滤层更像 small-only 放行器 / 避坑器，而不是自动切换器。
这就自然推到下一步：V1.2 不再硬切换，而是做增强开关。

V1.2 更新了什么思路

V1.1 已经说明：过滤层有价值，但不适合立刻做成完整的资产切换器。
所以 V1.2 的核心升级是：
不再强行 large → 300，而是保留 baseline，再用 small regime 做增强开关。

也就是说： - 默认仍然围绕 csi500 这个基线 - 但如果处于 small regime，就尝试切到更有弹性的 csi2000

V1.2 相对 V1.1 的修正

- 去掉了“large 必须做300”的强假设
- 把过滤层从“换挡器”降级为“增强器”
- 更接近真实可执行的策略表达

V1.2 重点验证什么

- small 环境是否应该切到 2000
- 非 small 环境是否继续保留 500 baseline
- 过滤层该做“切指数”还是只做“跳过”

V1.2 对策略表达的启发

V1.2 给出的不是“只做 small”的极端过滤器， 而是一个更自然的第一版实盘表达：

默认做 csi500；如果处于 small regime，就增强成 csi2000。

这比 V1.1 的 hard filter 更平衡： - 保留了 baseline 的交易覆盖 - 又利用了 small 环境下更高的弹性
